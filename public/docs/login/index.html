The Docs: Login

Let's get started

Logging in without code

To login just using this site, navigate to /login

Logging in with code

In order to login, a POST request must be made to /login, with the body containing either your username and password (named "username" and "password" in the body), or your token (named "token" in the body).

If a request contains both, the username and password will be given priority and the token will be ignored.

On a successful login, a new token will be generated and the old one will expire and be considered invalid.

This token will act as your credentials while you are using the site, as this will be the only form of identification stored on the client's computer.

Do NOT store the password anywhere. Only tokens and usernames may be stored on a device.
